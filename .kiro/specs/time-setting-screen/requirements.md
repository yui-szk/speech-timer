# 要件定義書

## 概要

時間設定画面は、ユーザーがタイマーの時間を直感的に設定できる機能です。現在のFigmaデザインに基づいて、メインタイマーの時間設定と1,2,3令の時間設定を提供します。各時間設定では、分と秒の数字部分をスクロールすることで時間を設定できるインターフェースを提供します。

## 要件

### 要件 1

**ユーザーストーリー:** ユーザーとして、画面上部でメインタイマーの時間を設定したい。そうすることで、主要なタイマー時間を直感的に調整できる。

#### 受け入れ基準

1. 時間設定画面が表示される時、システムは画面上部中央に大きなメインタイマー時間表示を配置する
2. メインタイマー時間が表示される時、システムはMM:SS形式で時間を表示する
3. ユーザーが分の数字部分をスクロールする時、システムは分の値を0-59の範囲で変更する
4. ユーザーが秒の数字部分をスクロールする時、システムは秒の値を0-59の範囲で変更する
5. 時間設定中、システムは現在設定中の時間を視覚的にハイライトする

### 要件 2

**ユーザーストーリー:** ユーザーとして、画面下部で1,2,3令の時間を設定したい。そうすることで、複数の段階的なタイマー時間を設定できる。

#### 受け入れ基準

1. 時間設定画面が表示される時、システムは画面下部に1,2,3令の時間設定エリアを配置する
2. 各令の時間が表示される時、システムはMM:SS形式で時間を表示する
3. ユーザーが各令の分の数字部分をスクロールする時、システムは該当する令の分の値を0-59の範囲で変更する
4. ユーザーが各令の秒の数字部分をスクロールする時、システムは該当する令の秒の値を0-59の範囲で変更する
5. 各令の時間設定は独立して操作可能である

### 要件 3

**ユーザーストーリー:** ユーザーとして、スクロール操作で時間を設定したい。そうすることで、直感的で正確な時間入力ができる。

#### 受け入れ基準

1. ユーザーが分の数字をスクロールする時、システムは滑らかなスクロールアニメーションを提供する
2. ユーザーが秒の数字をスクロールする時、システムは滑らかなスクロールアニメーションを提供する
3. スクロール中、システムは現在の値を中央に配置し、前後の値を薄く表示する
4. スクロールが停止する時、システムは最も近い有効な値にスナップする
5. 分と秒のスクロールは独立して操作可能である

### 要件 4

**ユーザーストーリー:** ユーザーとして、各令のベル通知設定を制御したい。そうすることで、どの段階で音声アラートを受け取るかを選択できる。

#### 受け入れ基準

1. 各令の時間設定の下に、システムはベルトグルボタンを配置する
2. ベルトグルが有効化される時、システムは視覚的インジケーター（アクティブ状態のベルアイコン）を表示する
3. ベルトグルが無効化される時、システムは視覚的インジケーター（非アクティブ状態のベルアイコン）を表示する
4. ベルが有効な令のタイマーが完了する時、システムは設定されたベル音を再生する
5. ベルが無効な令のタイマーが完了する時、システムは音を再生しない

### 要件 5

**ユーザーストーリー:** ユーザーとして、時間設定画面からメインタイマー画面に遷移してタイマーを開始したい。そうすることで、設定した時間でタイマーを実行できる。

#### 受け入れ基準

1. 時間設定画面が表示される時、システムは画面下部に再生ボタンを配置する
2. ユーザーが再生ボタンを押下する時、システはメインタイマー画面に遷移する
3. メインタイマー画面への遷移時、システムは設定されたメインタイマー時間を適用する
4. メインタイマー画面への遷移時、システムは設定された各令の時間とベル設定を適用する
5. 時間設定画面がアプリケーションの初期画面として表示される

### 要件 7

**ユーザーストーリー:** ユーザーとして、メインタイマー画面で時間の直接編集ができないようにしたい。そうすることで、時間設定は専用画面で行う明確な操作フローを維持できる。

#### 受け入れ基準

1. メインタイマー画面が表示される時、システムは時間表示を読み取り専用として表示する
2. ユーザーがメインタイマー画面の時間表示をタップしても、システムは編集モードに入らない
3. 時間設定の変更が必要な場合、ユーザーは時間設定画面に戻る必要がある
4. メインタイマー画面では、システムはタイマーの開始/停止/リセット機能のみを提供する

### 要件 6

**ユーザーストーリー:** ユーザーとして、時間設定画面がアプリの視覚デザインと一致してほしい。そうすることで、一貫したユーザー体験を得られる。

#### 受け入れ基準

1. 時間設定画面が表示される時、システムはメインアプリと同じカラースキーム（ティールグラデーション背景）を使用する
2. 時間値を表示する時、システムはメインアプリと一貫したNoto Sansフォントファミリーを使用する
3. アクティブ/選択状態を表示する時、システムはプライマリテキストカラー（#2c6975）を使用する
4. 非アクティブ/非選択状態を表示する時、システムはライトカラー（#a6d5cd）を使用する
5. インターフェースを表示する時、システムはメインタイマー画面と同じスペーシングとレイアウトパターンを維持する
